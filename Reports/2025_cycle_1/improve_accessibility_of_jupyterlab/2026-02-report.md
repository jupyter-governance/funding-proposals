## Summary of Work Completed

**Initial Research:** Isabel Paredes began by reviewing previous keyboard accessibility work and existing blog posts regarding Jupyter's navigation improvements.

**Issue Review:** A review was conducted on several reported accessibility violations, specifically JupyterLab Issues #9399, #16890, and #16891.

**Community Coordination:** Attended two Jupyter Contributor meetings to establish a plan for addressing the most urgent accessibility barriers.

**Manual Testing:** Performed a manual audit of the latest JupyterLab version to identify navigation gaps. Key issues found include:


* The help panel cannot be scrolled using arrow or page keys.


* Side panels and the file browser do not receive focus automatically when opened.


* Tab keys fail when editing text files.


* Focus outlines for buttons and checkboxes are often invisible or lack sufficient contrast.


* Specific UI elements, like the extension search box and kernel list items, are unreachable via keyboard.


**Development Activity:** Started work on Jupyter PR #18362 to address historical review issues and began making the Keyboard Shortcuts dialog navigable (Issue #18374).


## Deliverables/Milestones

* **Keyboard Navigation Overhaul:** The project is on track to ensure the UI is fully navigable, prioritizing core components like the settings editor.

* **User Testing Partnership:** Collaboration has been established with Université Paris Saclay's Pôle Handicap to identify blockers for students in scientific fields.
